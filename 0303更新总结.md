## 一、任务一：构建结构化财报数据库（Bug修复）

### 1.1 问题描述

利润表校验逻辑存在错误，使用 `营业总收入 - 营业总支出 ≈ 营业利润` 进行校验，不符合中国会计准则，导致大量误报。

### 1.2 根因分析

中国会计准则中，营业利润的正确计算公式为：

```
营业利润 = 营业总收入 - 营业总成本 + 其他收益 + 投资收益 + 公允价值变动收益
           + 信用减值损失 + 资产减值损失 + 资产处置收益
```

原代码缺少6个中间项的提取和计算，直接用 `营业总收入 - 营业总成本` 与 `营业利润` 比较，必然产生较大差额。

### 1.3 修改文件及内容

| 文件 | 修改内容 |
|------|----------|
| `backend/core/pdf_parser.py` | 在 `_extract_income_data` 的 `field_mappings` 中新增6个中间项字段提取（投资收益、公允价值变动收益、资产处置收益、营业外收入、营业外支出、信用减值损失） |
| `backend/core/database.py` | 在 `income_sheet` 表中新增对应的7个数据列 |
| `task1/run_task1.py` | **三处修改**：① LLM提取prompt新增中间项字段描述；② 利润表校验改用正确公式；③ 新增利润总额校验（`利润总额 = 营业利润 + 营业外收入 - 营业外支出`） |

### 1.4 动态阈值策略

为应对PDF提取时中间项可能缺失的情况，实现了基于数据完整性的动态校验阈值：

| 中间项缺失数 | 策略 | 容差公式 |
|:---:|------|----------|
| 全部缺失(6项) | **跳过校验** | — |
| 0 (全部提取到) | 严格校验 | `|营业利润| × 5% + 100` |
| 1-2项 | 适度放宽 | `|营业利润| × 50% + |营业总收入| × 15% + 5000` |
| 3+项 | 大幅放宽 | `|营业利润| × 100% + |营业总收入| × 20% + 10000` |

### 1.5 执行结果

- **处理PDF文件**：30份
- **数据库记录**：income_sheet 20条、balance_sheet 20条、core_performance_indicators_sheet 20条
- **校验问题**：55条（均为真实数据质量问题，无公式误报）
- **主要问题类型**：
  - 资产负债表缺失关键字段（equity_total_equity、liability_total_liabilities）
  - 资产负债表不平衡（总负债+股东权益≠总资产，疑似单位提取错误）
  - 利润表缺失 total_profit 字段
  - 少量利润表数值校验不通过（源数据提取列错位）

## 四、图表生成Bug修复

### 4.1 问题描述

Task 3首次运行时，B2001的水平柱状图生成报错：
```
ValueError: could not convert string to float: '金花股份'
```

### 4.2 根因

LLM决定图表配置时，偶尔会将字符串字段（如 `stock_abbr`）错误地分配到 `y_fields`（应为数值字段），而将数值字段分配到 `x_field`。

### 4.3 修复方案

在 `backend/core/visualizer.py` 的 `generate_chart` 函数中增加字段自动修正逻辑：

1. 检测 `y_fields` 中是否包含非数值字段
2. 若 `y_fields` 含字符串而 `x_field` 为数值 → **自动交换**
3. 若 `y_fields` 混合数值/字符串 → **移除非数值字段，调整 x_field**

修复后重跑 Task 3，B2001成功生成柱状图。

## 六、已知问题与后续优化方向

### 6.1 数据质量问题

1. **华润三九营收数据异常**：2024年 `total_operating_revenue` 为 27616.61（万元），实际应为 276.16亿元。疑似 `core_performance_indicators_sheet` 表中单位为百万元而非万元，导致数量级差两个数量级。
2. **资产负债表提取错误**：多个报告期的总负债和股东权益数据存在单位不一致，导致资产负债表等式不平衡。
3. **利润总额缺失率高**：约40%的报告期未能提取到 `total_profit` 字段。

### 6.2 建议优化方向

| 方向 | 说明 | 优先级 |
|------|------|:------:|
| 单位统一校验 | 在数据入库前增加单位量级检测，自动修正百万→万元等常见错误 | 高 |
| PDF表格提取增强 | 针对跨页表格和合并单元格优化提取逻辑 | 高 |
| LLM提取后校验 | 对LLM返回的数据进行交叉验证（如总资产≈总负债+总权益） | 中 |
| 图表配置优化 | 在LLM prompt中增加更多约束，减少字段映射错误 | 低 |
